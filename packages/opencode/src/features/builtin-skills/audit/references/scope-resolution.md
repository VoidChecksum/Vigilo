# Scope Resolution Reference

Detailed scope resolution logic for Phase 0 of the audit workflow.

---

## Scope Resolution Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                    SCOPE RESOLUTION FLOW                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Check scope.txt (or audit-scope.txt, .scope)               │
│     │                                                           │
│     ├─[EXISTS]→ Parse scope file → Extract file list           │
│     │           → Proceed with in-scope files only              │
│     │                                                           │
│     └─[NOT EXISTS]→ 2. Check README.md                         │
│                        │                                        │
│                        ├─[HAS SCOPE SECTION]→                  │
│                        │   • Parse scope from README            │
│                        │   • Create scope.txt in project root   │
│                        │   • Proceed with extracted scope       │
│                        │                                        │
│                        └─[NO SCOPE INFO]→ 3. Auto-detect       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## Step 1: Check Existing scope.txt

```
Read("scope.txt")           # Primary
Read("audit-scope.txt")     # Alternative
Read(".scope")              # Hidden file
```

**If scope file exists:**
- Parse file paths (one per line, supports glob patterns)
- Filter comments (lines starting with `#`)
- Validate paths exist
- Store resolved scope to `.vigilo/scope.txt`

**Scope file format example:**
```
# In-scope contracts
src/core/Vault.sol
src/core/Oracle.sol
src/periphery/*.sol

# Out of scope (do not audit)
# src/mocks/**
# test/**
```

---

## Step 2: Check README.md for Scope

**If no scope.txt found, check README.md:**

```
Read("README.md")
```

**Look for scope indicators:**
- Sections titled: "Scope", "Audit Scope", "In Scope", "Files in Scope"
- Tables with file paths or contract names
- Lists under headers like "## Contracts" or "## Files to Review"
- Code4rena/Sherlock contest format scope sections

**Common README scope patterns:**

```markdown
## Scope
| Contract | SLOC | Purpose |
|----------|------|---------|
| src/Vault.sol | 200 | Main vault logic |
| src/Oracle.sol | 150 | Price feed |
```

```markdown
## Files in Scope
- `src/core/Vault.sol`
- `src/core/Oracle.sol`
- `src/periphery/Router.sol`
```

**If scope found in README:**
1. Extract file paths from the scope section
2. **Create scope.txt in project root** with extracted paths
3. Log: "Created scope.txt from README.md scope section"
4. Proceed with extracted scope

**Example scope.txt generation:**
```bash
# Auto-generated from README.md
# Generated by vigilo audit on ${DATE}

src/core/Vault.sol
src/core/Oracle.sol
src/periphery/Router.sol
```

---

## Step 3: Auto-detection (Last Resort)

**Only if BOTH scope.txt AND README.md have no scope info:**

```
# Solidity projects
Glob("src/**/*.sol")
Glob("contracts/**/*.sol")

# Rust/Anchor projects
Glob("programs/**/*.rs")

# Cairo projects
Glob("**/*.cairo")

# Move projects
Glob("**/*.move")
```

**Exclude common non-scope patterns:**
- `test/**`, `tests/**`
- `script/**`, `scripts/**`
- `mocks/**`, `mock/**`
- `node_modules/**`
- `lib/**` (Foundry dependencies)

**After auto-detection:**
1. Create scope.txt with detected files
2. **WARN user**: "No scope.txt found. Auto-detected {N} files. Please verify."
3. Proceed with auto-detected scope

---

## Step 4: Scope Validation

**Before proceeding, validate scope:**

```
For each file in scope:
    If file does NOT exist:
        WARN: "Scope file not found: {path}"
        Remove from active scope

    If file is in excluded patterns (test/, mock/):
        WARN: "File appears to be test/mock: {path}"
        Keep but flag for review
```

**Output**: Final scope stored at `.vigilo/scope.txt`

---

## Iron Law: SCOPE FIRST

```
❌ WRONG: Glob("**/*.sol") then analyze everything
❌ WRONG: Skipping scope check because "it's a small project"
❌ WRONG: Ignoring scope.txt and analyzing all files anyway

✅ RIGHT: Check scope.txt → README.md → Auto-detect (in order)
✅ RIGHT: Create scope.txt if extracting from README
✅ RIGHT: Only analyze files explicitly in scope
```
